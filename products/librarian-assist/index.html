Librarian Assist

Everything a small library needs to run — books, members, lending, events, and finances — built for Kerala's libraries, offline-first, with optional Google Drive backup.

View more details ↗ Get it on Google Play

The idea

Kerala's small libraries already run on paper registers — an in-out ledger, a members book, an event diary. Librarian Assist doesn't try to replace that workflow with something unfamiliar; it digitizes the exact registers librarians already keep, so the app feels obvious on day one instead of requiring retraining.

What we built

01

Books & members

Full catalogue and member directory, with bulk Excel import in the standard KSLC data-entry format — including automatic column matching if headers don't line up exactly.

02

In-out register

Lending and returns tracked with serial numbers, call numbers, and member details — month-by-month browsing and Excel export.

03

Calendar & holidays

Weekly and one-off holidays; lending automatically pauses on closed days.

04

Events

Library events with photos, contests, sub-events, and participant lists — exported to Word with the library's letterhead.

05

Finance & fees

Income, expenses, and membership fee collections, tied back into the library's books.

06

Backup & restore

Fully offline by default; optional Google sign-in backs up and restores data via the librarian's own Drive.

Under the hood

The app is offline-first by design — a local SQLite database is the source of truth, so a library with unreliable internet loses nothing. Excel import/export uses the exact column layout Kerala State Library Council already expects, with fuzzy header matching for sheets that don't match exactly. Backup is opt-in: when enabled, the app writes directly to a folder it creates in the librarian's own Google Drive, scoped to files it created — we never see the data ourselves.
